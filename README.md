# 超合議制将棋（MVP）

みんなで1局の将棋を指す、**1日1手の投票制Webサービス**です。  
その日の最多得票の合法手を次の一手として採用します。

## 1. ディレクトリ構成（提案 + 実装）

```text
super-consensus-shogi/
├── app.py                # Flaskアプリ本体（ルーティング/DB/将棋ロジック）
├── requirements.txt      # 必要パッケージ
├── README.md             # セットアップ手順・仕様
├── templates/
│   ├── index.html        # トップページ
│   ├── history.html      # 履歴ページ
│   └── admin.html        # 管理ページ
└── static/
    └── style.css         # 最小スタイル
```

## 2. 必要パッケージ一覧

- Flask
- python-shogi

`requirements.txt` に記載済みです。

## 3. 実装計画（短縮版）

1. SQLiteスキーマ作成（対局・指し手・投票・設定）
2. python-shogiで局面生成し、合法手のみ候補表示
3. 1端末1票（Cookieトークン）で投票登録
4. ランキング表示と手動集計（管理ページ）
5. 締切時刻設定と自動進行（リクエスト時チェック）
6. 履歴と局面スナップショット表示

## 4. MVP機能

- 現在盤面表示
- 現在手番表示
- 候補手への投票
- 投票ランキング表示
- 「今すぐ集計して1手進める」
- 履歴表示（手数とその時点の盤面）
- 締切時刻（時・分）設定

## 5. セットアップ手順

### 5-1. インストール

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 5-2. 起動

```bash
python app.py
```

ブラウザで以下を開きます。

- トップ: `http://localhost:5000/`
- 履歴: `http://localhost:5000/history`
- 管理: `http://localhost:5000/admin`

初回アクセス時に `shogi.db` が自動作成されます。

## 6. cron運用（本番想定）

MVPでは、リクエスト時に締切超過を検知して自動進行を試みます。  
本番では cron で管理処理を叩く構成を推奨します（例: 毎日0時）。

例（将来的に `curl` で管理エンドポイントを呼ぶ想定）:

```cron
0 0 * * * curl -s http://127.0.0.1:5000/admin
```

> 現状のMVPは管理画面の「今すぐ集計」ボタンを使うのが確実です。

## 7. 今の実装で未対応の点

- 厳密な終局理由表示（詰み・千日手などの詳細表示）
- 投票締切をまたぐ厳密なラウンド開始時刻管理
- 複数同票時の高度なタイブレーク（現状はUSI文字列昇順）
- 管理画面の認証
- 複数対局の同時運用

## 8. 次に追加するとよい機能（優先度順）

1. 管理画面の認証（最低限のパスワード保護）
2. cron専用の安全な集計エンドポイント追加（トークン付き）
3. 千日手・入玉・詰みなど終局ステータス表示
4. 投票候補の絞り込みUI（駒種/移動元/移動先検索）
5. 対局アーカイブ（過去ゲーム一覧と再生）

## 9. ラズベリーパイにデプロイする最小手順

1. Raspberry PiにPython3を用意
2. プロジェクトを配置し、仮想環境を作成
3. `pip install -r requirements.txt`
4. `python app.py` で起動確認
5. 常駐化する場合は `systemd` サービス化
6. 日次進行は cron + 管理操作（将来的に専用エンドポイント化推奨）
